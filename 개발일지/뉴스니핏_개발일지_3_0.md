## 2. 기술 스택 및 구조

### 2.1 전체 자동화 흐름
```
매일 오전 7시
GitHub Actions 자동 실행
→ 연합뉴스 RSS 수집 (주요·경제·정치 피드)
→ Gemini가 40-50대 중요 뉴스 5개 필터링
→ AI 요약 + 용어설명 + 원문링크 생성
→ 기존 뉴스 news_archive로 이동
→ Supabase news 테이블에 새 뉴스 5개 저장
→ GitHub Pages (index.html) 자동 업데이트
```

### 2.2 기술 스택

| 영역 | 기술 | 비용 |
|------|------|------|
| 뉴스 수집 | RSS 피드 크롤링 (연합뉴스) | 0원 |
| AI 필터링·요약 | Gemini API (gemini-2.5-flash) | 유료 (하루 몇 원) |
| 데이터베이스 | Supabase | 0원 (무료 플랜) |
| 자동화 | GitHub Actions (매일 오전 7시) | 0원 |
| 웹 배포 | GitHub Pages | 0원 |
| 앱 프레임워크 | React Native + Expo | 0원 |

### 2.3 Supabase DB 스키마

**news 테이블** (오늘 뉴스 5개만 유지)
```sql
CREATE TABLE news (
  id SERIAL PRIMARY KEY,
  created_at TIMESTAMP DEFAULT NOW(),
  category TEXT, title TEXT, headline TEXT,
  point1 TEXT, point2 TEXT, point3 TEXT,
  impact TEXT, source TEXT, url TEXT,
  term TEXT, term_desc TEXT
);
```

**news_archive 테이블** (지난 뉴스 보관)
```sql
CREATE TABLE news_archive (
  id SERIAL PRIMARY KEY,
  created_at TIMESTAMP DEFAULT NOW(),
  category TEXT, title TEXT, headline TEXT,
  point1 TEXT, point2 TEXT, point3 TEXT,
  impact TEXT, source TEXT, url TEXT,
  term TEXT, term_desc TEXT,
  briefing_date DATE
);
```

## 4. 핵심 기능

### 4.1 자동화 엔진 (index_final.js)
- RSS 수집: 연합뉴스 주요·경제·정치 피드 (피드별 5개씩 총 15개 수집)
- Gemini 필터링: 40-50대에게 중요한 뉴스 5개 자동 선별
- AI 요약: 완전한 문장으로 핵심 3줄 요약
- 실생활 영향: "나에게 어떤 의미인지" 구체적 해석 (대출이자 월 X만원 등)
- 용어 설명: 어려운 단어 1개 자동 추출 + 쉬운 설명
- 원문 링크: RSS에서 실제 기사 URL 추출
- 재시도 로직: 503 오류 시 30초 후 최대 5회 자동 재시도
- 아카이브: 매일 기존 뉴스를 news_archive로 이동 후 새 뉴스 저장

### 4.2 웹 앱 (index.html) — GitHub Pages 배포
- Supabase에서 실시간 뉴스 데이터 로드
- 로그인·게스트·잠금 Freemium 구조
- 카드 3개 무료 + 상세 1회 무료 → 가입 유도
- 카카오톡 공유 (뉴스니핏 브랜드 자동 삽입)
- 글씨 크기 3단계 조절
- 알림 시간 설정 모달
- 검증 배지·난이도 배지·카테고리 배지
- 모바일 반응형 (폰 프레임 제거)
- 용어 설명 탭하면 펼쳐짐
- 원문 기사 링크 연결

### 4.3 React Native 앱 (App.js)
- Supabase에서 실시간 뉴스 로드
- 홈·상세·저장·설정 화면
- 카카오톡 공유
- 글씨 크기 조절
- 읽음 진행률 바
- ※ 아직 Expo 프로젝트 연결 필요
