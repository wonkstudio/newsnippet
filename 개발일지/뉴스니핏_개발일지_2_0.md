## 1. 사업 기획

### 1.1 시장 기회
- 국내 40–50대 스마트폰 사용자 약 1,800만명
- 기존 뉴스 요약 서비스(뉴닉·어피티)는 2030 타겟 — **40–50대 특화 앱 부재**
- 구독자 1만명 달성 시 월 매출 약 3,900만원 (앱스토어 수수료 전)

### 1.2 타겟 페르소나

**페르소나 A — 직장인 김부장 (48세)**
- 출퇴근 시간 5분 안에 핵심 뉴스 파악 원함
- 경제·부동산·주식 정보에 민감
- 긴 기사 읽기에 피로감 높음

**페르소나 B — 자영업 이사장 (52세)**
- 정치·사회 흐름이 사업 매출에 직결
- 신뢰할 수 있는 출처 중요시
- 복잡한 정책 이슈를 쉽게 이해하길 원함

## 3. UI/UX 디자인

### 3.1 디자인 원칙 (40–50대 최적화)
- **색상:** 네이비(#1C2B4A) 헤더 + 하늘색(#EAF3FB) 배경 — 신뢰감 + 눈에 편안함
- **글씨:** 크고 굵게 — 폰트 크기 3단계 조절 가능 (작게·보통·크게)
- **카드:** 여백 넉넉하게 — 한 눈에 파악
- **언어:** "~했습니다" 경어체 — 2030 앱과 차별화
- **"AI 요약" 표현 제거:** "핵심 정리"로 대체 — 40–50대 AI 거부감 해소

### 3.2 화면 구성
```
로그인 화면
├── 카카오톡으로 시작하기
└── 맛보기 (카드 3개·상세 1회 무료)

홈 화면
├── 헤더 (앱 이름·날짜·진행률 바)
├── 뉴스 카드 목록
│   ├── 카테고리 배지 (경제/정치)
│   ├── 난이도 배지 (쉬움/보통/심화)
│   ├── 검증 배지 (★ 검증)
│   ├── 카드 번호 (1/5)
│   ├── 제목
│   ├── 3줄 요약 (완전한 문장으로)
│   └── 자세히 보기 · 카톡 공유 버튼
└── [게스트] 잠금 배너 + 가입 유도

상세 화면
├── 헤더 (뒤로가기·공유·저장)
├── 핵심 정리 (1·2·3 완전한 문장)
├── 나에게 어떤 의미? (실생활 구체적 영향)
├── 용어 설명 (탭하면 펼쳐짐)
├── 원문 기사 보기 (실제 링크)
└── 카카오톡으로 공유하기

저장 화면
└── 북마크한 뉴스 목록

설정 화면
├── 글씨 크기 (작게·보통·크게)
├── 관심 분야 선택
├── 알림 시간 설정 (시·분 모달)
└── 구독 플랜 정보
```

### 3.3 Freemium 접근 전략

## 4. 기술 스택

### 4.1 전체 구조
```
매일 오전 7시
GitHub Actions 자동 실행
→ 연합뉴스 RSS 수집 (주요·경제·정치)
→ Gemini가 40-50대 중요 뉴스 5개 필터링
→ AI 요약 + 용어설명 + 원문링크 생성
→ Supabase DB 저장
→ newsnfit.html 자동 업데이트
```

### 4.2 기술 스택 상세

| 영역 | 기술 | 비용 |
|------|------|------|
| 뉴스 수집 | RSS 피드 크롤링 (연합뉴스 주요·경제·정치) | 0원 |
| AI 필터링 | Gemini API — 40-50대 중요 뉴스 선별 | 유료 |
| AI 요약 | Gemini API — 완전한 문장으로 요약 | 유료 |
| 데이터베이스 | Supabase (DB·인증·스토리지) | 0원 |
| 자동화 | GitHub Actions (매일 오전 7시) | 0원 |
| 웹 배포 | Netlify | 무료 플랜 |
| 앱 프레임워크 | React Native + Expo (SDK 54) | 0원 |
| 푸시 알림 | Firebase FCM | 0원 |
| 앱스토어 등록 | 구글 플레이 $25 일회성 | 약 3.5만원 |

### 4.3 Supabase DB 스키마

```sql
CREATE TABLE news (
  id SERIAL PRIMARY KEY,
  created_at TIMESTAMP DEFAULT NOW(),
  category TEXT,
  title TEXT,
  headline TEXT,
  point1 TEXT,
  point2 TEXT,
  point3 TEXT,
  impact TEXT,
  source TEXT,
  url TEXT,
  term TEXT,
  term_desc TEXT
);
```

## 5. 핵심 기능

### 5.1 자동화 엔진 (index_final.js)
- **RSS 수집:** 연합뉴스 주요·경제·정치 피드에서 뉴스 수집
- **Gemini 필터링:** 40-50대에게 중요한 뉴스 5개 자동 선별
- **AI 요약:** 완전한 문장으로 핵심 3줄 요약
- **실생활 영향:** "나에게 어떤 의미인지" 구체적 해석
- **용어 설명:** 어려운 단어 1개 자동 추출 + 쉬운 설명
- **원문 링크:** RSS에서 실제 기사 URL 추출
- **재시도 로직:** 503 오류 시 30초 후 최대 5회 자동 재시도
- **Supabase 저장:** 매일 기존 데이터 삭제 후 새 데이터 저장

### 5.2 웹 프로토타입 (newsnfit.html)
- Supabase에서 실시간 뉴스 데이터 로드
- 로그인·게스트·잠금 로직
- 카카오톡 공유 (뉴스니핏 브랜드 자동 삽입)
- 글씨 크기 3단계 조절
- 알림 시간 설정 모달
- 검증 배지·난이도 배지·카테고리 배지
- 모바일 반응형 (폰 프레임 제거)

### 5.3 GitHub Actions 자동화
```yaml
매일 오전 7시 (UTC 22:00) 자동 실행
→ index_final.js 실행
→ 뉴스 수집·요약·Supabase 저장
```
